The Need for Semantics in the Web

This is the classic picture of hypertext which used to be a part of talks in the days when we had to explain what hypertext is. The web is a set of nodes and links.
[Tim Berners-Lee 1994, http://www.w3.org/Talks/WWW94Tim/]
To a computer, then, the web is a flat, boring world devoid of meaning.
[Tim Berners-Lee 1994, http://www.w3.org/Talks/WWW94Tim/]
This is a pity, as in fact documents on the web describe real objects and imaginary concepts, and give particular relationships between them.
[Tim Berners-Lee 1994, http://www.w3.org/Talks/WWW94Tim/]
Adding semantics to the web involves two things: allowing documents which have information in machine-readable forms, and allowing links to be created with relationship values.
[Tim Berners-Lee 1994, http://www.w3.org/Talks/WWW94Tim/]
The Need for Semantics in the Web
This means that machines, as well as operating on the web information, can do real things. For example, a program could search for a house and negotiate transfer of ownership of the house to a new owner.
[Tim Berners-Lee 1994, http://www.w3.org/Talks/WWW94Tim/]

RDF serialization formats

  1. Turtle - compact, human-friendly format.
  2. N-Triples - very simple, easy-to-parse, line-based format that is not as compact as Turtle.
  3. N-Quads - superset of N-Triples, for serializing multiple RDF graphs.
  4. JSON-LD - JSON-based serialization.
  5. N3 or Notation3 - a non-standard serialization that is very similar to Turtle, but has some additional features.
  6. RDF/XML - an XML-based syntax that was the first standard format for serializing RDF.

Query Linked Data

SPARQL Example

			
PREFIX foaf: <http://xmlns.com/foaf/0.1/>
PREFIX card: <http://www.w3.org/People/Berners-Lee/card#>
SELECT ?homepage
FROM <http://www.w3.org/People/Berners-Lee/card>
WHERE {
card:i foaf:knows ?known .
?known foaf:homepage ?homepage .
}
